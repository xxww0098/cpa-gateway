Task 15 — executor/openai.go usage.PublishRecord integration
============================================================

Scope
-----
Modify ONLY executor/openai.go. Parse OpenAI usage on every request
(both Execute non-stream path and ExecuteStream SSE path) and fire-
and-forget a cliproxyusage.Record via PublishRecord so the SDK
default usage manager routes it to the registered UsagePlugin
(sdk/usage.go). The plugin reads SettleCtx off ctx to decide
whether to settle.


Changes applied
---------------
executor/openai.go
  + import cliproxyusage "github.com/router-for-me/CLIProxyAPI/v7/sdk/cliproxy/usage"

  Execute (non-stream):
    - capture startedAt at entry
    - ParseOpenAIUsage(payload) after body read
    - expose parsed tokens under Response.Metadata["usage"]
      (non-SDK callers keep their existing compatibility surface)
    - call e.publishUsage(...) on BOTH success and failure paths
    - failing responses (4xx/5xx) publish with Failed=true +
      Fail.StatusCode + Fail.Body (truncated to 4 KiB)

  ExecuteStream:
    - capture startedAt at entry
    - promote opts to streamOpts with Stream=true so the record
      carries the same flag the request actually used
    - on pre-stream 4xx/5xx: publish a failed record immediately,
      then return upstreamStatusError (matches Claude/Codex path)
    - during streaming: accumulate raw SSE body in a bytes.Buffer
    - on ctx cancel / read error / EOF: publish via
      publishStreamUsage which runs parseOpenAIStreamUsage over
      the accumulated bytes

  New helpers (openai-local, mirroring codex/claude/gemini pattern):
    openAIRequestedModel(req, opts) → upstream model string
    (*OpenAICompatibleExecutor).publishUsage(...)
      — assembles & fires cliproxyusage.PublishRecord
    (*OpenAICompatibleExecutor).publishStreamUsage(...)
      — parses SSE body then delegates
    parseOpenAIStreamUsage(body) (UsageTokens, bool)
      - fast path: plain JSON body via ParseOpenAIUsage
      - SSE path: scan every `data: {json}` line, try
        ParseOpenAIUsage on each, keep the last successful parse
        (skips `data: [DONE]`). This matches OpenAI's streaming
        contract where the penultimate event carries the aggregate
        `usage` envelope when stream_options.include_usage=true.
    truncateOpenAIFailureBody(payload) string    — 4 KiB cap for
      Fail.Body


Record field population
-----------------------
  Provider    = e.provider    (ProviderOpenAI = "openai")
  Model       = openAIRequestedModel(req, opts)
  Alias       = cliproxyusage.RequestedModelAliasFromContext(ctx)
  Source      = e.provider
  RequestedAt = startedAt
  Latency     = time.Since(startedAt)
  Failed      = statusCode >= 400 || streamErr != nil
  AuthID      = auth.ID
  AuthIndex   = auth.Index
  AuthType    = auth.Provider
  Detail      = { Input, Output, Reasoning, Cached,
                  TotalTokens=Input+Output }
  Fail        = { StatusCode, Body (truncated to 4 KiB) } —
                only when failed
