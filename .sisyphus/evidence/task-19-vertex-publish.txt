Scope

- Pattern mirrored from Task 17 (Gemini), as specified — Vertex's
  usageMetadata envelope shares the Gemini shape, so ParseVertexUsage
  is reused directly for both non-stream and streamed responses.

Integration points
------------------
1. Execute (non-stream):
   - Captures requestedAt before the upstream call.
   - On success/failure, runs ParseVertexUsage(payload) and calls
     publishUsageFromPayload → cliproxyusage.PublishRecord(ctx, rec).
   - Failure path preserves the upstreamStatusError return; PublishRecord
     is still called (Failed=true, Fail.StatusCode=status).

2. ExecuteStream (streaming):
   - Chunks are forwarded to the client unchanged (no latency added).
   - Each chunk is simultaneously appended to an in-goroutine buffer and
     scanned with extractLatestVertexUsage. The scanner handles both
     SSE-framed `data: {...}` events and concatenated JSON objects,
     keeping the most recent usageMetadata parse.
   - If the per-chunk heuristic never succeeds, the aggregated buffer is
     re-parsed once with ParseVertexUsage as a fallback before publishing.
   - Publishing happens in a deferred-style finish block after the
     SSE/JSON stream drains, so streaming remains non-blocking.

3. Errors / context cancellation:
   - ctx.Err() path marks Failed=true; StatusCode is 0 for cancellations.
   - Non-EOF upstream read errors also mark Failed=true.
   - HTTP error responses read before chunking begins publish Failed=true
     with the concrete status code.
