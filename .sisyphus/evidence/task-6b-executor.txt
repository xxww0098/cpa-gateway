Task 6b evidence — executor/ package extraction
================================================

Scope
-----
- Removed five root-level executor_*.go files (package main) and consolidated their
  logic into package executor/ (openai.go, claude.go, gemini.go, codex.go, vertex.go)
  along with a shared executor/util.go for provider identifiers, ProviderConfig,
  upstreamStatusError, ApproximateTokensFromBytes, CopyOutboundHeaders,
  SanitizedProxyHeaders and the JSON metadata helpers.
- Updated handler_proxy.go to construct executor types via the new package
  (executor.NewOpenAICompatibleExecutor / NewClaudeExecutor / NewGeminiExecutor /
  NewCodexExecutor / NewVertexExecutor) and reference executor.ProviderOpenAI etc.
  plus executor.ApproximateTokensFromBytes / SanitizedProxyHeaders.
- Added exported helpers on executor.VertexExecutor (BaseURL, ServiceAccountJSON)
  and exposed executor.VertexMetadataServiceAccount / executor.CodexMetadataAccessToken
  so main-package callers can seed auth records without reaching for unexported fields.
- Preserved root-package shared helpers (stringFromMap / nestedString /
  stringFromJSONString / stringFromJSONBytes) in a new metadata_util.go because
  handler_sdk_mgmt.go still depends on them. Semantics match the original
  implementation previously living in executor_claude.go.
