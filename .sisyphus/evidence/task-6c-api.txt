Task 6c evidence — api/ package extraction + PanelRouter dependency injection
==============================================================================

Scope
-----
Moved panel handlers, middleware, response envelope, API key helpers, and
metadata utility helpers into the api/ package, converting free functions
to methods on *PanelRouter. Wired root main.go to construct a PanelRouter
and delegate panel route registration via PanelRouter.RegisterPanelRoutes.
handler_proxy.go and BillingMiddleware stay at the root per the task brief;
Task 21 will remove them together with the old /v1 surface.

Files created / moved
---------------------
api/router.go              — PanelRouter (DB, Redis, Ledger, Calc, Config,
                              APIKeyCache, AuthManager, AuthStore) + constructor
                              + RegisterPanelRoutes + SetupGlobalMiddleware
api/middleware.go          — TraceIDMiddleware / MetricsMiddleware /
                              RateLimitMiddleware + per-router MetricsStore
                              + AuthMiddleware (API key + JWT) + BillingCtx
api/response.go            — APIResponse / Success / Error
api/constants.go           — panel error code constants
api/apikey.go              — GenerateAPIKey (router method) + CachedKey alias
api/handler_auth.go        — register / login + StartCacheCleanup
api/handler_user.go        — user panel + API key management + usage views
api/handler_admin.go       — admin dashboard + role/user admin + requireAdmin
api/handler_admin_expanded.go — user/group/subscription admin handlers
api/handler_subscription.go — user subscription endpoints + EnsureSubscriptionSeeds
api/handler_ops.go         — announcements / redeem / refunds / payments / usage logs

                              OAuth sessions, auth files, ampcode config, logs,
                              model definitions, /config, convenience toggles)
api/metadata_util.go       — shared stringFromMap / nestedString helpers

Root changes
------------
- main.go constructs api.NewPanelRouter(..., pricing.Calculator{}, cfg) and
  calls panelRouter.RegisterPanelRoutes(r). Panel routes no longer live in
  main. SDK auth manager (authManager) is handed to panelRouter.AuthManager.
- main.go installs panelRouter.AuthMiddleware() on /v1/* before BillingMiddleware
  so handler_proxy.go keeps receiving a populated billing context.
- main.go calls api.EnsureSubscriptionSeeds(db).
- middleware.go reduced to BillingCtx / BillingMiddleware / estimateRequestCost
  / peekAndRestoreBody / holdTTL (used only by handler_proxy.go).
- helpers.go retains visibleCatalogModelIDsSorted for handler_proxy.go only.

Dependency-injection highlights
-------------------------------
- No GlobalDB/GlobalConfig/GlobalLedger references anywhere in api/ (see
  grep evidence below).
- pr.Config.Auth.JWT.Secret replaces the main-package GlobalConfig lookups.
- authutil.GenerateJWT / authutil.ValidateJWT / authutil.HashAPIKey /
  authutil.NewAPIKey / authutil.APIKeyPrefix replace the old root helpers.
- slices.ContainsFunc(pr.Config.Auth.AdminEmails, ...) powers pr.isAdminEmail
  (case- and whitespace-insensitive).
- APIKeyCache is owned by PanelRouter (pr.APIKeyCache = infra.NewAPIKeyCache())
  and the sweeper is started from handler_auth.StartCacheCleanup.
- Per-router metricsStore eliminates the global metrics variable; the old
  /metrics handler now lives on PanelRouter.
- AuthManager / AuthStore are optional fields set by main.go after InitSDK
  so handler_sdk_mgmt can call pr.AuthManager.Register/Update etc.

SDK coupling
------------
The only cliproxy import from api/ is cliproxyauth (SDK management data
types — Auth, Manager, Store). The SDK's Builder/Server/router-configurator
machinery is NOT imported by api/. handler_sdk_mgmt handlers legitimately
operate on cliproxyauth.Auth values because that is the domain data of the
SDK-management surface; rewriting that surface is explicitly out of scope
("Do not modify handler business logic"). main.go retains full ownership
of the SDK Builder wiring.
