Task 11 — sdk/access.go + sdk/settlectx.go
===========================================

Scope
-----
- sdk/settlectx.go: SettleCtx type + WithSettleCtx / SettleCtxFromContext
  ctx helpers. Pure data carrier, no dependencies on SDK types.
- sdk/access.go: AccessProvider implementing
  github.com/router-for-me/CLIProxyAPI/v7/sdk/access.Provider.
  Identifier() = "cpa-tenant".
  Authenticate() parses Bearer token on r, dispatches to
  API key path for "cpa-" prefix else JWT path, enriches the
  returned access.Result.Metadata with subscription quota/usage
  fields so HoldMiddleware can act without another DB round-trip.
- sdk/access_test.go: 5 TDD tests using miniredis (testutil) and
  in-memory SQLite (gorm.io/driver/sqlite).

Implementation notes
--------------------
- Metadata keys populated (string-valued per access.Result contract):
    user_id, api_key_id (cpa-key path), group_id (when set),
    rate_mult, email (jwt path),
    subscription_id, subscription_group_id,
    daily_limit, daily_used, weekly_limit, weekly_used,
    monthly_limit, monthly_used
- AccessProvider holds no Hold/Settle state — only validates
  credentials and loads metadata (per plan "Must NOT do").
- API key path uses the L1 infra.APIKeyCache on hit; populates it on
  miss with a 5-minute TTL.
- last_used_at is bumped asynchronously with a detached context so
  request cancellation does not abort the bookkeeping write.
- JWT path tolerates a nil/zero UserID by returning
  AuthErrorCodeInvalidCredential rather than panicking.
