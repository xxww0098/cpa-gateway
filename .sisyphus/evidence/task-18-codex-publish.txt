Integration Summary
-------------------
- Added import cliproxyusage "github.com/router-for-me/CLIProxyAPI/v7/sdk/cliproxy/usage"
- Execute: captures startedAt, parses ParseCodexUsage(payload) after
  io.ReadAll, exposes tokens via Response.Metadata["usage"], and calls
  publishUsage on both success and upstream 4xx/5xx paths.
- ExecuteStream: captures startedAt, publishes a failed record on
  pre-stream 4xx/5xx, accumulates SSE bytes in a bytes.Buffer inside
  the streaming goroutine, and calls publishStreamUsage on ctx cancel /
  read error / EOF.
- Helpers added (codex-local):
    (*CodexExecutor).publishUsage(ctx, auth, tokens, parsed, failed,
      status, payload, startedAt)
      — assembles & fires cliproxyusage.PublishRecord
    (*CodexExecutor).publishStreamUsage(ctx, auth, body, failed,
      status, startedAt) — parses SSE body then delegates
    parseCodexSSEUsage(body) (UsageTokens, bool)
      — fast path: plain JSON body via ParseCodexUsage
      — SSE path: walk every `data: {json}` line, keep the last
        successful parse (matches the terminal-chunk-carries-usage
        contract of OpenAI/Codex stream_options.include_usage=true)
    truncateCodexFailureBody(payload) string — 4 KiB cap for Fail.Body

Record field population

  Provider    = providerCodex    ("codex")
